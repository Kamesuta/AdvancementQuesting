# AdvancementQuesting プロジェクト仕様書

## プロジェクト概要

FTB Quest・Better Questing・HQM などのクエスト MOD と同様の機能を持つ Minecraft プラグイン。
プレイヤーがクエストを受注し、達成条件をクリアして報酬を得られるシステムを提供する。

### 特徴

- ブラウザで動くマップベースのクエストエディタ（PC・スマホ対応）
- 編集権限を持たないプレイヤーによるクエスト提案・投票システム
- Minecraft の Advancement システムとの統合
- クエストデータは JSON ファイルで管理（Git 差分が読める）

---

## システム構成

```
┌──────────────────────────────────────────────────────────┐
│  ブラウザ (React SPA)                                      │
│  クエストマップエディタ / 進捗確認 / 提案・投票             │
└────────────────────┬─────────────────────────────────────┘
                     │ REST API  /api/*
         ┌───────────┴───────────┐
         │                       │
┌────────▼────────┐    ┌─────────▼────────────┐
│   Mock Server   │    │  Minecraft Plugin     │
│ Express+SQLite  │    │  Paper / Java         │
│ (開発・テスト)   │    │  組み込み HTTP サーバー │
└─────────────────┘    └──────────────────────┘
                                 │
                       ┌─────────┴────────────┐
                       │                      │
                  JSON ファイル           SQLite DB
                  (クエスト定義)          (進捗・セッション)
```

### 本番構成

- ビルド済み静的ファイル (`dist/`) をプラグインの組み込み HTTP サーバーから配信
- API もプラグイン内で実装 — 追加サーバー不要

### 開発構成

- フロントエンド: Vite dev server (`localhost:5173`)
- API: Mock サーバー (`localhost:3000`)
- Vite proxy で `/api/*` を Mock サーバーへ転送

## 機能一覧

### A. 認証

#### A-1. ログイン方法

**6桁コード認証（メイン）**
```
/quest code
→ 6桁の認証コードを生成 (有効期限: 5分)
→ Web ページでコードを入力
→ セッション確立、ロールをトークンに紐付け
```

**URL リンク認証（補助）**
```
/quest login
→ プレイヤー固有の URL を生成
→ チャットにクリック可能なリンクを表示
→ URL を開くと自動でセッション確立
```

#### A-2. 権限ロール

| ロール | 取得条件 | 主な操作 |
|--------|---------|---------|
| `player` | ゲームにログインしている全員 | クエスト閲覧・進捗確認・提案投稿・投票 |
| `editor` | OP または権限ノード `aq.editor` | クエスト作成・編集・提案の承認/却下 |
| `admin` | OP | editor の全権 + クエスト削除 |

ロールはセッション発行時にプラグインが決定し、Web 側は再判定しない。

---

### B. クエストエディタ（Web）

マップベースのビジュアルエディタ。`editor` / `admin` ロールのみ操作可能。

#### B-1. クエストデータ構造

```
クエスト:
  - ID (自動生成)
  - タイトル
  - サブタイトル
  - 説明文
  - アイコン (Minecraft アイテム ID)
  - タグ (複数)
  - 前提クエスト (複数)
  - タスク (達成条件) — 詳細は B-2
  - 報酬 / 報酬テーブル参照 — 詳細は B-3
  - 公開状態 (draft / public / hidden)
```

#### B-2. タスク種別

| タイプ | 説明 | 必要パラメータ |
|--------|------|-------------|
| `item` | アイテムを集める | `itemType`, `count` |
| `advancement` | Advancement を達成する | `advancementId` |
| `checkmark` | 管理者が手動でチェック | `value`(確認メッセージ) |
| `command` | コマンド実行で達成 | `command` |

#### B-3. 報酬種別

| タイプ | 説明 | 必要パラメータ |
|--------|------|-------------|
| `item` | アイテム付与 | `itemType`, `count` |
| `experience` | 経験値付与 | `amount` |
| `command` | コマンド実行 | `command` |
| `permission` | 権限ノード付与 | `node` |
| `money` | お金付与 (Vault) | `amount` |

クエストは個別報酬の代わりに**報酬テーブル**を参照することもできる。
報酬テーブルは複数クエストで共有可能な報酬セット。

#### B-4. マップエディタ操作

| ツール | マウス操作 | タッチ操作 |
|--------|-----------|-----------|
| 選択・移動 | ドラッグでノード移動、背景ドラッグでパン | スワイプでパン、ノードドラッグで移動 |
| クエスト追加 | クリックで追加 | タップで追加 |
| 依存関係 | ノードをクリック→クリックで接続 | ノードをタップ→スワイプして離す |
| クエスト編集 | ノードをクリックでモーダル | タップで全画面編集 |
| 削除 | ノードをクリックで削除 | タップで削除 |

### C. クエスト提案システム

編集権限を持たないプレイヤーが、クエストの追加を申請する仕組み。

#### フロー

```
1. player がエディタと同じ UI でクエスト内容を作成
2. 「提案として送信」でサーバーへ保存
3. 全プレイヤーに通知（MC チャット + Web バッジ）
4. プレイヤーが 👍/👎 で投票
5. editor / admin が内容を確認して承認 or 却下
6. 承認時: クエスト JSON として書き出し → 即公開
   却下時: 却下理由を添えてプレイヤーに通知
```

#### 提案の制約

- マップ座標は含まない（承認後にエディタで配置）
- 提案内容は `questSnapshot` として丸ごと保存（承認時にそのまま使用）
- 1プレイヤー1票（反対方向で投票すると上書き、同方向で再投票すると取り消し）

### D. プレイヤー向け Web 機能

#### D-1. クエストマップ（閲覧）

- 全クエストの配置図（エディタと同じビュー、編集不可）
- ノードの色でクエスト状態を表示:

| 色 | 状態 |
|----|------|
| グレー | 未解放（前提未達成） |
| 青 | 受注可能 |
| 黄 | 進行中 |
| 緑 | 達成済み |

- クリック/タップで詳細表示

#### D-2. クエスト詳細

- 達成条件一覧（チェックマーク付き）
- 報酬プレビュー
- 前提クエスト
- 「報酬を受け取る」ボタン（完了後のみ有効）

#### D-3. 進捗確認

- 受注中クエスト一覧
- 達成済み一覧
- 各条件の達成状況

---

### E. Minecraft 内機能

#### E-1. 進捗システム統合

- クエストの `advancement` タスクは Minecraft の Advancement と紐付く
- Advancement 達成時にプラグインが自動で進捗を更新
- カスタム Advancement を自動生成してクエスト専用タブに追加

#### E-2. クエストマップ（床面表示）

- Web エディタのマップを Minecraft の床面に投影
- クエストアイコンをマップエンティティで表示
- 右クリック: クエスト詳細 GUI
- 左クリック: チェスト GUI（タスク・報酬・受取ボタン）

#### E-3. 通知システム

| タイミング | 通知方法 |
|-----------|---------|
| クエスト解放 | タイトル表示 + サウンド |
| 条件達成 | アクションバー（例: `条件2達成! 3/5`） |
| クエスト完了 | タイトル + サブタイトル + 花火 |
| 新規提案 | チャット通知（クリックで Web に飛べるリンク付き） |

#### E-4. コマンド

```
プレイヤー:
  /quest               — Web URL を表示
  /quest code          — 6桁認証コード取得
  /quest login         — ログイン URL 取得
  /quest progress      — 進捗一覧

管理者:
  /quest reload        — 設定・クエストデータの再読み込み
  /quest map create    — マップ配置を設定
  /quest give <player> <questId>   — クエスト強制付与
  /quest reset <player>            — プレイヤー進捗リセット
  /quest approve <proposalId>      — 提案承認
  /quest reject <proposalId> [理由] — 提案却下
```
